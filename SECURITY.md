# Security Policy

## Supported Versions

This repository contains documentation and guidelines. Security updates apply to the documentation itself to ensure no sensitive information is exposed.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it responsibly:

### What to Report

- Exposed credentials, API keys, or secrets in any skill
- Personal or sensitive information in documentation
- Malicious code patterns in examples
- Security anti-patterns being recommended as best practices

### How to Report

**Do NOT create a public issue for security vulnerabilities.**

Instead:

1. **Email the maintainers directly** (see README for contact)
2. **Use GitHub Security Advisory**: 
   - Go to the Security tab
   - Click "Report a vulnerability"
   - Provide detailed information

### What to Include

- Description of the vulnerability
- Location (file path and line numbers)
- Potential impact
- Suggested fix (if you have one)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Assessment**: Within 7 days
- **Fix**: Depends on severity
  - Critical: Within 24-48 hours
  - High: Within 7 days
  - Medium: Within 30 days
  - Low: Next release cycle

## Security Best Practices

This repository follows these security practices:

- No credentials or secrets in any files
- No personal or sensitive information
- Security-focused code review for all PRs
- Regular audits of skill content for security anti-patterns

## Disclosure Policy

- Security issues will be addressed privately
- Once fixed, a security advisory will be published
- Credit will be given to reporters (if desired)
